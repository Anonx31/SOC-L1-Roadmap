# Network Protocols Notes

## Introduction

Network protocols are rules and standards that allow devices to communicate over a network. Understanding these protocols is essential for SOC Analysts because network traffic analysis is a key part of incident investigation and threat detection.

---

# HTTP (HyperText Transfer Protocol)

## Purpose

HTTP is used to transfer web pages and web content between clients and servers.

## Default Port

* TCP 80

## Characteristics

* Unencrypted
* Human-readable traffic
* Commonly used for websites

## Example

A user visits a website and sends an HTTP GET request to retrieve a webpage.

## SOC Relevance

SOC Analysts may inspect HTTP traffic to identify:

* Suspicious websites
* Malware communication
* Data exfiltration attempts

---

# HTTPS (HyperText Transfer Protocol Secure)

## Purpose

HTTPS provides secure communication between clients and web servers.

## Default Port

* TCP 443

## Characteristics

* Encrypted using TLS/SSL
* Protects confidentiality and integrity
* Widely used on modern websites

## Example

Logging into online banking securely.

## SOC Relevance

Analysts can examine:

* Destination domains
* TLS certificates
* Connection metadata

Even though the content is encrypted.

---

# DNS (Domain Name System)

## Purpose

DNS translates domain names into IP addresses.

## Default Port

* UDP 53
* TCP 53

## Example

When a user visits github.com, DNS resolves the domain name to an IP address.

## SOC Relevance

DNS analysis can reveal:

* Malware command-and-control domains
* Suspicious lookups
* Domain generation algorithms (DGAs)

---

# ICMP (Internet Control Message Protocol)

## Purpose

ICMP is used for network diagnostics and error reporting.

## Common Commands

```bash
ping google.com
```

## Example

Echo Request and Echo Reply messages generated by ping.

## SOC Relevance

Analysts monitor ICMP traffic for:

* Network reconnaissance
* Host discovery
* Unusual tunneling activity

---

# TCP (Transmission Control Protocol)

## Purpose

TCP provides reliable communication between devices.

## Characteristics

* Connection-oriented
* Reliable
* Ordered packet delivery

## Three-Way Handshake

1. SYN
2. SYN-ACK
3. ACK

## Common Ports

* HTTP (80)
* HTTPS (443)
* SSH (22)

## SOC Relevance

Understanding TCP sessions helps analysts investigate:

* Network connections
* Suspicious communications
* Lateral movement

---

# UDP (User Datagram Protocol)

## Purpose

UDP provides fast communication without guaranteed delivery.

## Characteristics

* Connectionless
* Faster than TCP
* No handshake

## Common Uses

* DNS
* VoIP
* Video streaming

## SOC Relevance

Analysts monitor UDP traffic for:

* DNS abuse
* Amplification attacks
* Unusual network activity

---

# DHCP (Dynamic Host Configuration Protocol)

## Purpose

DHCP automatically assigns IP addresses to devices on a network.

## Ports

* UDP 67
* UDP 68

## DHCP Process

1. Discover
2. Offer
3. Request
4. Acknowledge

(DORA Process)

## SOC Relevance

DHCP logs help analysts identify:

* New devices joining a network
* Rogue devices
* Network inventory changes

---

Understanding network protocols is a fundamental skill for SOC Analysts. Protocol analysis helps identify malicious activity, investigate incidents, and understand network behavior during security events.
