# Project 2 - Brute Force and Account Lockout Investigation

## Project Overview

This project simulates a brute-force attack scenario against a Windows user account and investigates the resulting security events using Windows Event Viewer.

The objective was to generate multiple failed login attempts, trigger an account lockout, analyze the relevant Windows Security Event IDs, and document the investigation process.

---

## Objectives

* Generate failed login attempts
* Trigger a Windows account lockout
* Investigate Windows Security Logs
* Analyze authentication-related Event IDs
* Build an incident timeline
* Create a SOC-style investigation report

---

## Lab Environment

### Operating System

* Windows 10

### Tools Used

* Event Viewer
* Windows Security Logs
* Command Prompt

### Test Account

* targetuser

---

## Attack Simulation

A local Windows account was created and targeted with multiple incorrect login attempts.

The repeated authentication failures triggered the Windows account lockout policy, preventing further access until administrative intervention occurred.

After unlocking the account, a successful login was performed to complete the investigation scenario.

---

## Event IDs Investigated

### Event ID 4625

**Description:** Failed Login Attempt

**Purpose:**
Records unsuccessful authentication attempts.

**Key Findings:**

* Multiple failed login attempts detected.
* Authentication failures occurred within a short period.
* The targeted account was identified.

---

### Event ID 4740

**Description:** Account Locked Out

**Purpose:**
Records account lockout events generated by security policy enforcement.

**Key Findings:**

* Account lockout successfully triggered.
* Security controls prevented additional login attempts.

---

### Event ID 4624

**Description:** Successful Login

**Purpose:**
Records successful authentication activity.

**Key Findings:**

* Successful login recorded after account recovery.
* User access restored following administrative action.

---

## Investigation Process

### Phase 1

* Created test account.
* Generated failed login attempts.

### Phase 2

* Investigated Event ID 4625.
* Counted failed authentication attempts.

### Phase 3

* Investigated Event ID 4740.
* Confirmed account lockout.

### Phase 4

* Unlocked account.
* Generated successful login event.

### Phase 5

* Investigated Event ID 4624.
* Built incident timeline.
* Documented findings.

## Key Takeaways

This project demonstrated how Windows Security Logs can be used to identify brute-force activity and account lockout events.

By analyzing Event IDs 4625, 4740, and 4624, it was possible to reconstruct the sequence of events and understand how authentication activity is recorded within Windows environments.

The investigation provided practical experience with one of the most common alert types handled by SOC Analysts.
